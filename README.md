# RAG Search App

A full-stack Next.js application for searching, uploading, and viewing documents with Retrieval-Augmented Generation (RAG) capabilities. It integrates with Supabase for storage and vector search, and uses Cohere and Groq APIs for embeddings and LLM-powered answers.

---

## Features

- **Semantic Search:** Ask questions and get answers based on your uploaded documents.
- **Document Upload:** Upload PDFs, DOCX, and other supported files.
- **Document Management:** View, preview, and manage your uploaded documents.
- **PDF Viewer:** Inline PDF preview with text extraction.
- **Modern UI:** Built with React, Tailwind CSS, and Next.js App Router.

---

## Tech Stack

- **Frontend:** Next.js (App Router), React 19, Tailwind CSS
- **Backend:** Next.js API routes
- **Database & Storage:** Supabase (Postgres, Storage, Vector Search)
- **LLM & Embeddings:** Cohere (embeddings), Groq (LLM)
- **PDF Parsing:** pdf2json, mammoth (DOCX)
- **Other:** LangChain, TypeScript

## Getting Started

### Prerequisites

- Node.js 18+
- Supabase project (with vector extension enabled)
- API keys for Cohere and Groq

### Installation

1. **Clone the repo:**
	 ```bash
	 git clone <repo-url>
	 cd rag-search-app
	 ```

2. **Install dependencies:**
	 ```bash
	 npm install
	 # or
	 yarn install
	 ```

3. **Configure environment variables:**

	 Create a `.env.local` file with:
	 ```
	 NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
	 NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY=your_supabase_anon_key
	 SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
	 COHERE_API_KEY=your_cohere_api_key
	 GROQ_API_KEY=your_groq_api_key
	 ```

4. **Run the development server:**
	 ```bash
	 npm run dev
	 ```

	 Open [http://localhost:3000](http://localhost:3000) in your browser.

## Usage

### Search

- Enter a question in the search bar on the home page.
- The app will return an answer generated by the LLM, citing relevant document sources.

### Upload Documents

- Go to the "Documents" page.
- Click "Upload" to add new files (PDF, DOCX, etc.).
- Uploaded documents are chunked, embedded, and indexed for semantic search.

### View Documents

- Browse your uploaded documents.
- Click on a document to preview its content (PDFs are rendered inline).

## API Overview

- **POST /api/search**  
	Accepts `{ query: string }`. Returns `{ answer, sources }` using Cohere embeddings and Groq LLM.

- **GET /api/documents**  
	Lists all uploaded documents.

- **POST /api/upload**  
	Handles file uploads, chunking, and embedding.

---

## Customization

- **Styling:** Tailwind CSS is used for all styling. Edit `globals.css` or component classes.
- **Vector DB:** Supabase vector extension is required for similarity search.
- **LLM/Embeddings:** You can swap out Cohere/Groq for other providers by editing the API route logic.
